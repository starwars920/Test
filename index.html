De-bugging

Imagine yourself finishing up this extremely long and complicated piece of code but when you click Run, it doesn't work? It makes you want to punch the computer in the face, right? This means there is a mistake in your code. So to fix it, we do this thing called de-bugging. Did you know that the first computer error was a moth caught in the computer? That's where de-bugging comes from. Most coding languages use semicolons at the end of every line of code to tell the computer "Hey! I want you to run this!". If you are using a coding language you know uses a semicolon (for example Javascript), check every line of code to see if you forgot any semicolons. If your language doesn't use a semicolon, try just reading through all your code to check for possible mistakes. If even that won't work, grab another coder and find you mistake. When all else fails, google.

Variables

What is a variable? Well, you've probably heard of it in a mathy setting, x can equal any value you want to give it. Variables in coding are a lot like that. A variable is a sort of container that you can store information in inside your code. Using var you can tell your computer "Hey, I'm (insert age here) years old and I want you to store that information in the variable PinkFluffyUnicornsThatDanceOnRainbows!". Obviously you wanto a variable name that is short and easy to understand if your boss read your code. If you wanted to store an age, you might store it in the variable Age or YearsOld. You can also code your program to when the reader or player are asked a question, they can type in their answer and it will be stored for later. Very cool. But that's only the tip of the iceberg when it comes to coding.
